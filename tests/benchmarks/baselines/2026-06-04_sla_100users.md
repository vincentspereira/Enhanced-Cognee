# SLA load test -- 100 concurrent users (2026-06-04)

Capacity-focused Locust run of `tests/load/locustfile.py` against the HTTP
server (`src/enhanced_cognee_mcp.py`) on the default stack (Postgres 18 +
pgvector, Qdrant, Valkey, ArcadeDB). Dev-open auth (the locustfile sends no
auth headers), 100 users, spawn-rate 10, 90s, mixed Read/Write/Mixed profiles.

## SLA targets (from `tests/load/locustfile.py`)

| Target | Result | Status |
| --- | --- | --- |
| >= 100 RPS sustained | **210.6 RPS** | MET (2.1x) |
| 0% error rate at 100 users | **0.00%** (0 / 18,862 reqs) | MET |
| p95 latency < 200ms | **170ms aggregate** (every endpoint < 200ms) | MET |

## The fix that got p95 under SLA

The first run hit 191.9 RPS / 0% errors but **missed the p95 target at 230ms**,
driven entirely by the search path (`search_memories` p95 = 250ms, p50 = 100ms).
Root cause: `search_memory()` filters with
`to_tsvector('english', coalesce(content, '')) @@ plainto_tsquery(...)` but
there was **no matching full-text index** -- Postgres recomputed the tsvector
for every row on every query (bitmap/seq scan over ~6.5k rows).

Fix: added a functional GIN index matching the query expression exactly
(`docker/init-scripts/01-init-pgvector.sql`):

```sql
CREATE INDEX IF NOT EXISTS idx_documents_content_fts
    ON shared_memory.documents
    USING GIN (to_tsvector('english', coalesce(content, '')));
```

`EXPLAIN` confirms the planner switched to `Bitmap Index Scan on
idx_documents_content_fts`.

## Caveats on these numbers

- Single dev box (server + 4 DB containers + Locust all co-resident); a
  dedicated 2-core VPS with the load generator off-box would differ, very
  likely favourably for latency.
- Dev-open auth -- per-request JWT/API-key verification adds a small constant
  overhead not measured here (this run characterises raw capacity, matching how
  the provider-comparison baselines were captured).
- `search_memories` exercises the Postgres full-text path (no caller-supplied
  embedding), which is the dominant production read pattern for HTTP callers.

## Reproduce

```bash
# stack up (docker/docker-compose-enhanced-cognee.yml), server on :8080, then:
python -m locust -f tests/load/locustfile.py --headless \
  --host http://127.0.0.1:8080 --users 100 --spawn-rate 10 --run-time 90s \
  --csv tests/benchmarks/output/sla_100u --only-summary
```
